# 移动端 UI 布局

**状态：设计已确认，待实现。**

## 设计原则

- 面向单手操作，基准宽度 320-430px；关键操作触控目标至少 44px。
- 用 Vant 组件表达表单、弹层、列表、空态和错误态；保留上海房地产公共信息的克制、清晰视觉。
- 页面不复制原站桌面端密集表格；数据表在手机上改为卡片/横向可滚动区域。
- 首屏优先显示更新时间、数据来源和当前错误状态。

## 总体结构

```text
┌──────────────────────────────┐
│ 顶部品牌 / 页面标题 / 原站入口 │  sticky
├──────────────────────────────┤
│ 页面内容                       │  scroll
│  Hero / 查询入口               │
│  数据卡片 / 列表 / FAQ          │
│  loading / empty / error state │
├──────────────────────────────┤
│ 首页 新房 二手房 租赁 统计      │  fixed bottom tab
└──────────────────────────────┘
```

顶部不超过 56px；底部 Tab 考虑 `env(safe-area-inset-bottom)`。外链统一新窗口打开并在按钮旁标注“原站”。

## 交互契约

| 输入 | 作用域 | 行为 |
|---|---|---|
| 点击底部 Tab | 全局 | 路由切换，页面状态按路由 query 恢复 |
| 点击筛选 | 新房/二手房 | 打开底部筛选面板，焦点进入第一个字段 |
| `Enter` | 搜索输入框 | 提交当前查询；如果需验证码，先创建验证码会话 |
| 点击验证码图 | 验证码弹层 | 刷新验证码；旧 session 失效 |
| 点击提交 | 验证码弹层 | 校验非空后提交筛选；失败保留筛选值 |
| `Esc`/返回 | 弹层 | 关闭且不应用未确认筛选 |
| 点击分页 | 列表 | 只改变 page，重新请求；不会自动抓取下一页 |
| 点击原站 | 所有页面 | 打开固定 fallback URL，不把当前未知参数拼入 URL |

## 状态变体

- **Loading**：Vant skeleton/card 占位；按钮进入 loading，禁止重复提交。
- **Empty**：显示“暂无可展示数据”和“调整筛选/打开原站”。
- **Error**：显示错误码对应用户文案、重试按钮和原站按钮；`UPSTREAM_BLOCKED` 明确说明不能由移动版代替原站验证。
- **Captcha**：底部弹层覆盖页面；图片、输入框、刷新按钮、提交按钮；不会把 sessionId 显示为可复制文本。
- **Busy**：提交时保留已输入筛选条件，显示“正在查询”，不自动轮询。
- **Constrained**：320px 宽度下卡片单列；统计表允许横向滚动；长标题两行截断并在详情显示全文。

## 组件树

```text
AppShell
├─ TopBar
├─ RouterView
│  ├─ HomePage
│  ├─ NewHousePage
│  │  ├─ FilterToolbar
│  │  ├─ HouseList
│  │  └─ CaptchaDialog
│  ├─ OldHousePage
│  │  ├─ FilterToolbar
│  │  ├─ HouseList
│  │  └─ CaptchaDialog
│  ├─ LeasePage
│  └─ TradePage
└─ BottomTabBar
```

共享 `ApiState<T>` 包含 `idle/loading/success/empty/error`；组件不直接调用 `fetch`。所有上游文本先进入结构化字段，公告正文需要专门清洗/转纯文本后才能显示。

## 页面入口

- `/` 首页
- `/new-house` 新房列表
- `/new-house/:id` 新房详情
- `/old-house` 二手房列表
- `/old-house/:id` 二手房详情
- `/lease` 租赁 FAQ/资料
- `/trade` 统计
- `/notice/:id` 公告/政策/要闻详情（若上游 schema 验证通过）
